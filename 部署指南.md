# 部署前准备清单

## 你需要准备的 4 样东西

### ✅ 1. GitHub 账号（如果没有）
- 访问 https://github.com 注册

### ✅ 2. GitHub Personal Access Token
按下面步骤生成：

1. 登录 GitHub → 点击右上角**头像** → **Settings**
2. 左侧最下面 → **Developer settings**
3. 左侧 → **Personal access tokens** → **Tokens (classic)**
4. 点击 **Generate new token** → **Generate new token (classic)**
5. **Note** 填：`cloudflare-deploy`
6. **Expiration** 选：`No expiration`
7. **Select scopes** 勾选 ✅ **`repo`**（其他不用选）
8. 滚动到底部点 **Generate token**
9. **复制 token**（页面关闭后无法再看到！）

⚠️ **保存好这个 token**，之后推送代码时会用到

---

### ✅ 3. 在 GitHub 创建新仓库

1. 访问 https://github.com/new
2. **Repository name**: `jinxin-resume`（或你喜欢的名字）
3. **Description**: 可选填"个人简历网站"
4. **Public**（公开，免费）
5. ⚠️ **不要勾选** Add a README file
6. ⚠️ **不要勾选** Add .gitignore
7. ⚠️ **不要选择** License
8. 点 **Create repository**

---

### ✅ 4. Cloudflare 账号
- 访问 https://dash.cloudflare.com 注册（建议用 GitHub 账号直接登录）

## 部署到 Cloudflare Pages

推送成功后：

1. 登录 https://dash.cloudflare.com
2. 左侧菜单 → **Workers & Pages**
3. 点击 **Create** → **Pages** 标签
4. 点击 **Connect to Git**
5. 选择 **GitHub** → 授权
6. 选择仓库 **`jinxin-resume`**
7. 配置：
   - **Project name**: `jinxin-resume`（默认域名会变成 `jinxin-resume.pages.dev`）
   - **Production branch**: `main`
   - **Build command**: 留空
   - **Build output directory**: 留空
8. 点 **Save and Deploy**
9. 等待 2-3 分钟，部署成功！

## 买自己的域名（可选）

如果想买 `jinxin.dev` / `jinxin.com` 这样的专属域名：

### 最省事：在 Cloudflare 买
1. https://dash.cloudflare.com → **Domain Registration** → **Register Domains**
2. 搜索你想要的名字 → 加入购物车 → 付款
3. 价格参考：
   - `.dev` 约 ¥80/年
   - `.com` 约 ¥80/年
   - `.xyz` 约 ¥10/年
   - `.top` 约 ¥10/年
4. 买完后 Cloudflare Pages → 你的项目 → **Custom domains** → 添加域名
5. Cloudflare 自动配置 DNS 和 SSL 证书

### 不在 Cloudflare 买（其他域名商）
阿里云、腾讯云、Namecheap 等都行，DNS 解析指向 Cloudflare 的地址即可。

## 常见问题

**Q: 推送时提示 "Repository not found"**
A: 检查仓库名是否正确；确保仓库已创建

**Q: 推送时提示 "Authentication failed"**
A: 必须用 Personal Access Token，不是 GitHub 密码

**Q: Cloudflare 部署后打开是空白**
A: 等 1-2 分钟，Cloudflare 还在部署；检查 Build output directory 是否留空

**Q: 图片不显示**
A: 检查 `df681eb7-70c0-4c79-9253-58ead2b6544b.png` 是否在仓库根目录
